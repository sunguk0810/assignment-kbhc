# 이슈 및 궁금증 정리

### 1. [이슈] macOS Gradle 스크립트 불러오기 시 문제
- 상황
  - JDK 21이 설치되어 있음에도 불구하고 Gradle 스크립트 불러오기 시 JDK 17을 찾을 수 없다고 함.
    ```shell
    Cannot find a Java installation on your machine (Mac OS X 15.6.1 aarch64) matching: {languageVersion=17, vendor=any vendor, implementation=vendor-specific, nativeImageCapable=false}. Toolchain download repositories have not been configured.
    ```
    ```shell
    ❯ java -version
    openjdk version "21.0.2" 2024-01-16 LTS
    OpenJDK Runtime Environment Temurin-21.0.2+13 (build 21.0.2+13-LTS)
    OpenJDK 64-Bit Server VM Temurin-21.0.2+13 (build 21.0.2+13-LTS, mixed mode)
    ```
- 원인
  - Spring Boot 프로젝트 구성시 JDK 17을 사용하기로 설정
  - Gradle은 `build.gradle.kts` Java Toolchain에서 설정한 Java 버전을 사용
  - 따라서 macOS에서 설치된 JDK 21은 JDK 17이 아니기에 오류 발생
- 해결 방법
  - JDK 17 설치하여 빌드 문제 해결
- 참고 문서
  - [Toolchains for JVM Projects](https://docs.gradle.org/current/userguide/toolchains.html)

### 2. [이슈] 어노테이션에서 Enum 상수 사용 불가
- 상황
  ```java
    @JsonTypeInfo(
      use = JsonTypeInfo.Id.NAME,
      include = JsonTypeInfo.As.PROPERTY,
      property = "type"
    )
    @JsonSubTypes({
      @JsonSubTypes.Type(value = StepInfo.class, name = MeasureType.STEPS)
    })
    public interface HealthDetail {
    }
  ```
  - 측정타입의 걸음수로 name을 지정하여 사용하고 싶었으나 `Incompatible types. Found: 'com.github.sunguk0810.assignment.domain.health.constant.MeasureType', required: 'java.lang.String` 오류 발생
- 해결 방법
  - 내부 상수 클래스를 생성하여 Enum 클래스 아래에 있는 String 문자열을 바라보게 함.
